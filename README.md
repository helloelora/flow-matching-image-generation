# Flow matching for image generation

**Conditional Flow Matching vs. Diffusion Models: a systematic comparison**

> Implementation of Conditional Flow Matching (CFM) from scratch, benchmarked against DDPM and DDIM on CIFAR-10, with a focus on the trade-off between sampling speed (NFE) and generation quality (FID).

---

## Overview

Flow Matching is a recent generative modeling framework that learns a continuous vector field `v_theta(x, t)` to transport samples from noise (t=0) to data (t=1) along straight-line paths. Compared to diffusion models, which follow curved stochastic trajectories, the optimal-transport coupling in CFM defines direct interpolations:

```
x_t = (1 - t) * z_0 + t * x_1
```

where `z_0 ~ N(0, I)` is Gaussian noise and `x_1` is a data sample. A neural network is trained to regress the velocity `u = x_1 - z_0` at each point along the path. At inference, new images are generated by solving the ODE:

```
dx/dt = v_theta(x, t),    t in [0, 1]
```

using standard solvers (Euler, RK4). The straight-path formulation requires significantly fewer integration steps than diffusion models for comparable quality.

## Methods

### 1. Conditional Flow Matching (CFM)
- Trains a velocity field `v_theta(x_t, t)` via a simple MSE regression loss on the target velocity `(x_1 - z_0)`
- Generates samples by integrating the learned ODE from t=0 to t=1
- Supports Euler and RK4 solvers with variable step counts

### 2. DDPM (Baseline)
- Denoising Diffusion Probabilistic Model with 1000-step Markov chain
- Standard training with epsilon-prediction objective

### 3. DDIM (Baseline)
- Deterministic sampling from a pretrained DDPM
- Allows skipping steps (10, 20, 50, 100) without retraining

All three methods share the **same U-Net backbone** for fair comparison.

## Getting started

### Installation

```bash
git clone https://github.com/<your-username>/flow-matching-image-generation.git
cd flow-matching-image-generation
pip install -r requirements.txt
```

### Training

```bash
# Train CFM on CIFAR-10
python scripts/train.py --config configs/cfm_cifar10.yaml

# Train DDPM baseline
python scripts/train.py --config configs/ddpm_cifar10.yaml
```

### Sampling

```bash
# Generate samples with CFM (Euler, 10 steps)
python scripts/sample.py --config configs/cfm_cifar10.yaml --solver euler --steps 10

# Generate samples with DDIM (50 steps)
python scripts/sample.py --config configs/ddim_cifar10.yaml --steps 50
```

### Evaluation

```bash
# Compute FID score
python scripts/evaluate.py --generated results/samples/ --dataset cifar10
```
